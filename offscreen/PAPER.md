# A Decodable Signalling Code Co-evolves but Stays Private to Kin: Survival Selection, Clonal Descent, and Why No Public Code Bootstrapped in a Minimal World

**Chunxuan Yang** — Sogang University, Seoul, Republic of Korea — yang2023@sogang.ac.kr · yangchunxuan1@gmail.com

---

## Abstract

We ask whether a decodable signalling code can *grow* from random weights under survival selection alone—no gradient on the task, no oracle labels, no borrowed weights—and whether what grows is a public shared convention or a private kin code. In a minimal hand-coded 2-D lethal-vent world, tiny tied code-table agents evolve a decodable communication channel, and we decompose the result. Both arms carry the full code-table; the only manipulated variable is the reproduction-fitness signal (survival vs uniform-random, with the reproduction RNG byte-matched between arms). The tied-architecture-plus-clonal-descent baseline alone—the random-fitness control, which also fixates to ~one lineage—reaches mutual intelligibility (MII) 0.1174, already ~73× the 1/625 ≈ 0.0016 chance floor; at the pre-registered gen-28 fixation point survival selection adds a paired win-margin of **+0.0548** on top (95% CI [0.0234, 0.0849], n=48; survival arm MII 0.17219 vs random-fitness control 0.1174). Selection therefore *sharpens and tunes* a largely descent-determined channel rather than originating it from nothing. A secondary, post-hoc transient probe (n=24, checkpoints at gen 28/56/100/150) shows this margin is a **transient acceleration**, not a sustained equilibrium difference: the survival-arm MII stays ~stable (~0.16–0.17) while the random-fitness control's MII rises (0.117 → 0.138) as clonal fixation alone accumulates cross-agent decodability, so the paired margin decays from +0.0569 [0.0162, 0.1021] (gen 28, CI excludes 0) through +0.0489 [0.0151, 0.0821] (gen 56, excludes 0) to +0.0329 [−0.0014, 0.0687] (gen 100, includes 0) and +0.0249 [−0.0072, 0.0571] (gen 150, includes 0). Within-kin decodability is thus fundamentally a clonal-descent phenomenon that survival selection merely *accelerates*; the gen-28 positive stands at its timescale, and the transient finding is the empirical justification for scoping the headline to 28 generations. The code stays private to *kin*: the survival arm fixates to a single founder lineage (mean dominant-lineage share 99.1%, inverse-Simpson N_eff 1.02), so within-founder intelligibility runs ≈0.16 (~100× chance) while cross-founder intelligibility sits at the chance floor. Maintaining lineage diversity is necessary but not sufficient: in the one sustained-diversity cell (niching at pop=96, soft96, n=8) coexisting non-kin lineages still decode one another only at the floor. Three heterogeneous interventions (different designs and scaffolds, 40 vs 48 generations) that triangulate cross-lineage pressure all leave cross-founder intelligibility at chance; under these regimes a public cross-lineage code did not bootstrap (positive-or-inconclusive tier, not banked as a formal negative). The co-evolution headline is scoped to this world, this architecture, and a 28-generation timescale; the cross-lineage probes run 40–48 generations. Methodologically, we contribute a bidirectional self-audit whose single claim is **bidirectionality**—it catches deflation as well as inflation: in this project it caught a pilot/formal false *negative* and an RNG-offset bug in the headline control (whose fix left the conclusion intact and slightly strengthened the margin, +0.045 → +0.0548).

**Keywords:** artificial life; emergent communication; signalling code; survival selection; clonal descent; kin selection; adversarial self-audit methodology

## 1. Introduction

Where does a shared signalling code come from? A communication channel is *decodable* when
a receiver can recover a signal's referents from the signal alone, and a code is *public*
when distinct lineages — not just kin — can decode one another. Most computational accounts
of how such a code arises take one of two routes. In the signalling-game and reinforcement
tradition, communication is *trained in*: a channel is optimised by gradient descent on a
task with a differentiable channel (Foerster et al., 2016), or by reward on a referential
task that supplies a task signal each round (Lazaridou et al., 2017). In the iterated-learning
tradition, structure is *induced by design*: a transmission bottleneck is imposed between
generations so that only learnable, compressible codes survive (Kirby, Cornish & Smith, 2008). Both routes answer a real question, but neither asks the more basic one we take
up here: can a decodable signalling code **grow** — neither designed-in nor trained-in — from
random initial weights under **survival selection alone**, with no gradient on the communicative
task, no oracle or truth labels, and no borrowed or pretrained weights? And if such a code does
grow, is it a **public** code shared across the population, or a **private** code confined to kin?

We study this question in a deliberately minimal substrate: a hand-coded, fixed-rule 2-D
lethal-vent world in which agents metabolise, can die, and reproduce, and in which a tiny
tied code-table (the only communicative apparatus) is initialised from N(0,1) noise. The
world is not learned and the channel is not optimised; the sole pressure on communication is
that organisms that use it must, on average, survive and reproduce more. Mutual
intelligibility (MII) is measured not by weight similarity but as inter-agent codebook
agreement — a sampled set of referents is passed through one agent's emission and another's
decoding, and we score exact-tuple recovery, against a chance baseline of 1/625 ≈ 0.0016.
This minimality is the point: a small world makes the mechanism legible and lets us build a
control that is matched to the experimental arm in every respect except the reproduction-fitness
signal, so that any MII advantage we attribute to *selection for communication* is fairly earned
rather than assumed.

The scientific contribution is a single, scoped finding with three parts, and a decomposition
runs through all of it. First, within this world and over a 28-generation timescale, a decodable
channel **co-evolves from scratch** — but the result decomposes into a descent baseline and a
selection increment. The control here is not communication-ablated; both arms carry the full
`UnifiedIO` code-table, and the only manipulated variable is the reproduction-fitness signal
(survival vs uniform-random). That control — which we call the **random-fitness control** —
also fixates to ~one lineage, so its MII of **0.1174** (already ~73× the 1/625 chance floor) is
the tied-architecture-plus-clonal-descent baseline, not zero. At the pre-registered gen-28 fixation point, survival selection adds
**+0.0548** on top: the survival-selected arm reaches MII **0.17219** against the random-fitness
control's **0.1174**, a paired MII win-margin of **+0.0548, 95% CI [0.0234, 0.0849]** at n=48.
We therefore frame selection as *sharpening and tuning* a largely descent-determined channel,
not originating it from nothing, and the recovered intelligibility is semantic rather than an
artifact of weight overlap. A secondary, post-hoc transient probe (§5.1) shows that this gen-28
margin is a **transient acceleration** of clonally-shared decodability rather than a sustained
equilibrium difference: by ~gen 100 the random-fitness control's own clonal fixation reaches
comparable MII and the paired margin's CI includes zero. The gen-28 positive is the primary,
CI-clean result and stands at its timescale; the decay is why we scope the headline to 28
generations, and it reinforces that within-kin decodability is fundamentally a clonal-descent
phenomenon that selection merely accelerates. Second, this code is **not public**. The survival arm fixates to a
single founder lineage — mean dominant-lineage share **99.1%**, N_eff **1.02** — so the co-evolved
code is a **kin code**: at generations where lineages coexist, within-lineage intelligibility runs
~100× chance while cross-lineage intelligibility sits at the chance floor. Third, actively
maintaining lineage diversity (niching at larger population) is *necessary but not sufficient* —
in the one sustained-diversity cell (soft96, n=8) coexisting non-kin lineages still decode one
another only at the floor — and putting cross-lineage decoding directly under selection does not
bootstrap a public code either: higher pressure only crashes survival, because non-kin codes are
mutually undecodable and forcing non-kin listening starves foraging. We name this a
**pressure-vs-survival tension**. We state plainly what this is and is not: it is a mechanism
observed in *one world, one architecture, and one timescale*, not a universal law. Where a result
is a negative, we report its power tier in the same breath: the cross-lineage outcome is "a public
code did not bootstrap under the regimes tested (n=8, positive-or-inconclusive tier; not banked as
a formal negative — that would want n=16)." We claim that under these regimes a public code did not
bootstrap, never that one cannot.

The second contribution is methodological, and we frame it as a disciplined practice we recommend
rather than a co-equal scientific result. Emergent-communication work in artificial life and
multi-agent learning has a recognised vulnerability: claims of emerged communication built on
missing controls, on gradient or oracle leakage, or on single-seed positives. We present a reusable
self-audit — an architecture-matched control, a fair baseline, multi-seed bootstrap-CI win-margins,
frozen pre-registration, an explicit no-oracle redline, and a code-grounded adversarial review loop.
Its single methodological claim is **bidirectionality**: a self-audit should catch *deflation* as
readily as inflation. We demonstrate this with two in-scope catches reproducible from this paper's
own artifacts: (1) a pilot/formal false **negative** — an effect we had nearly discarded because a
formal treatment arm was compared against a pilot control — and (2) an RNG-offset bug in the
headline random-fitness control, surfaced by a multi-model adversarial review, whose fix left the
conclusion intact and slightly *strengthened* the margin (+0.045 → +0.0548). Catching a deflation is
the direction conventional self-checking almost never covers; that is the property we argue is worth
reusing.

Finally, we are explicit about scope. This is a self-contained study of a single arc in a
single minimal world; it is not a benchmark and not a generality claim. The result that a
decodable code here grows but stays kin-private *motivates* — rather than requires — a
richer world in which shared referents and sustainable diversity under communicative pressure
might let a public code emerge. That richer world is future work; the question we answer here
is self-contained, and we answer it honestly.

## 2. Related Work

Our study sits at the intersection of five lines of work — emergent communication in multi-agent learning, signaling-game accounts of meaning, the iterated-learning tradition on compositionality, the artificial-life and embodied-robotics literature on the evolution of communication, and kin selection / relatedness-and-signalling — and is shaped by a sixth: a methodological critique about over-claimed emergence and weak baselines. We position the contribution against each in turn, emphasizing what we do *not* assume.

**Emergent communication in MARL.** A large body of deep multi-agent reinforcement-learning work shows that communication protocols arise when agents are jointly optimized on a task, typically with a differentiable or otherwise privileged channel: agents back-propagate through messages, gradients passing through the channel between agents (differentiable inter-agent learning; Foerster et al., 2016), or are scored by reward on a referential task that supplies a task signal on every round (Lazaridou et al., 2017). Grounded compositional language can emerge in multi-agent populations under such task-optimized training (Mordatch & Abbeel, 2018), and the modern deep-emergent-communication literature is broad enough to have its own reviews and tooling (Lazaridou & Baroni, 2020; Kharitonov et al., 2019). These setups demonstrate that *gradient pressure plus a task signal* can shape a channel quickly, but the channel is, in an important sense, installed by the optimizer and the supervision. Our setup deliberately removes these affordances. There is no gradient on the channel — the only gradient anywhere in the substrate (an optional cultural-imitation step) is unused in the load-bearing arms — and no oracle: agents never receive truth labels for what a message "should" mean. Adaptation proceeds by mutation and selection on survival alone. The question is therefore not whether a protocol can be *trained in*, but whether a decodable signalling code can *grow* from random weights under selection — a strictly weaker set of assumptions than the MARL emergent-communication literature relies on.

**Signaling and convention games.** The game-theoretic tradition, from Lewis's signaling games (Lewis, 1969) to Skyrms's evolutionary treatment (Skyrms, 2010), establishes that stable signaling conventions can emerge from local dynamics without a designer fixing the code in advance, and that multiple incompatible conventions can be locally stable. This framework is the natural lens for our central finding — that a code which is decodable *within* a lineage need not be decodable *across* lineages — and our notion of a private kin code is consistent with the existence of mutually incompatible conventions. We differ in substrate and in what is at stake: classical signaling games abstract the sender, the referent, and the payoff into a matrix, whereas in our world the "signal" is the argmax of a tiny evolved network, the "referent" is a state of a hand-coded 2-D physics, and the "payoff" is metabolic survival. We thus offer a mechanistic instance of the convention-multiplicity that signaling theory predicts in the abstract, and we measure inter-agent codebook agreement directly rather than assuming a shared equilibrium.

**Kin selection and signalling among relatives.** Our headline outcome is a *kin*-private code, which places it squarely in the kin-selection tradition: relatedness lowers the threshold for cooperative and communicative traits to be favoured (Hamilton, 1964). In our world a private code is selectively cheap precisely because the survival population collapses to ~one founder lineage, so almost every receiver an agent meets is kin; there is little fitness reason for codes to align across lineages an agent will, in practice, rarely encounter. We make this relatedness structure explicit and measurable (N_eff, dominant-lineage share) rather than assumed, and read the kin-privacy result through this lens.

**Iterated learning and compositionality.** A prominent account of how structured, transmissible language arises is iterated learning, in which a *transmission bottleneck* — learners exposed to only a subset of the data — pressures languages toward compositional, generalizable structure (Kirby, Cornish & Smith, 2008). Neural iterated learning realizes this pressure in deep agents and can yield compositional structure (Ren et al., 2020). At the same time, emergent-language compositionality is not automatically tied to generalization, and agent languages can be effective without being human-like or compositional (Kottur et al., 2017; Chaabouni et al., 2020). Our design takes a pointed stance toward this tradition: we do **not** impose a transmission bottleneck. Rather than building in the mechanism known to produce shareable structure, we ask whether one is *needed* — whether survival selection alone, absent any imposed bottleneck, yields a code that travels across lineages. Our scoped result (within this world, this architecture, and a 28-generation timescale) leans toward "no": a public cross-lineage code did not bootstrap under the regimes tested, which we read as motivating — not proving — the case that an iterated-learning bottleneck or an ecological analogue of one may be required. We are careful not to over-read this: it is a no-detectable-effect result under the regimes tested, not a demonstration that the architecture *cannot* cross the gap.

**Artificial-life and embodied evolved communication.** Within ALife, the evolution of communication has been studied through simulated agents under selection (Cangelosi & Parisi, 2002) and through situated, grounded language-game and robotic models in which embodied agents evolve or negotiate communication (Steels, 2003; Quinn, 2001; Marocco & Nolfi, 2007). Our work is squarely in this lineage in spirit — minimal, mechanistic, selection-driven, embodied in a metabolic world — but is distinguished by an explicit, architecture-matched control that manipulates only the reproduction-fitness signal, and by treating decodability as a quantity to be *measured* (via a referent→emit→decode→exact-tuple mutual-intelligibility probe against a stated chance floor) rather than asserted from observed behavior.

**The replication-hygiene critique, and why a methodology contribution.** A recurring concern across emergent-communication and ALife research is that "language emerged" claims are often confounded by gradient or oracle leakage, lack a matched control, and rest on underpowered single-seed runs; relatedly, emergent protocols are often degenerate or fail without explicit pressure (Kottur et al., 2017; Chaabouni et al., 2020). This critique motivates our second contribution, framed as a disciplined practice we recommend: a reusable adversarial self-audit combining an architecture-matched control, a fair baseline, multi-seed bootstrap-CI win-margins, frozen pre-registration, a no-oracle redline, and code-grounded adversarial review. Its single methodological claim is **bidirectionality** — it is built to catch *deflation* as readily as inflation — demonstrated within this very project by its catching a false *negative* (a pilot/formal control mismatch) and an RNG-offset bug in the headline control that, once fixed, slightly strengthened rather than weakened the result.

**Positioning.** The field reliably produces public, grounded, or compositional communication, but in lanes that supply exactly what our setting withholds: gradient-MARL with task rewards and interacting partners (Mordatch & Abbeel, 2018; Lazaridou & Baroni, 2020), a deep-learning infrastructure for emergent-language games (Kharitonov et al., 2019), imposed iterated-learning bottlenecks (Ren et al., 2020), or populations of large language models that already possess a borrowed, pre-trained language (Flint Ashery et al., 2025). The LLM line is a contrast case only, not a mechanism we build on. Our question is therefore not whether public communication can be engineered — it can — but the origins/sufficiency question those lines do not address: whether a decodable, grounded signalling code can grow from random weights under survival selection alone, with no gradient on communication, no oracle, and no borrowed weights. Within that constraint our scoped finding (decodable but kin-private; no public bootstrap; a transient within-kin advantage) both instantiates and is delimited by kin-selection theory (Hamilton, 1964) and the iterated-learning account of structure (Kirby et al., 2008; Ren et al., 2020).

## 3. World and Agents

The substrate is deliberately minimal and entirely hand-coded. Nothing in the world or the agent body is learned; the only adaptive object in the whole study is each agent's communication code-table, and even that adapts by mutation and selection rather than by any gradient on the survival task. This section describes the world, the agent, and the reproduction loop, and states the from-scratch invariants that make the later claims fair.

### 3.1 The RTC lethal-vent world

The world (`rtc_world.py`, with constants frozen in `config/prereg_rtc.py`, `RTC_VERSION = "RTC-MVP0-rich-world-v1"`, freeze date 2026-06-28) is a 48×48 grid (`RTC_GRID = 48`) carrying a 4-channel scalar field (`RTC_K = 4`). The field evolves by a fixed-rule physics: advection (`RTC_ADVECT_SPEED = 0.45`), diffusion (`RTC_DIFFUSE_ALPHA = 0.035`), per-step decay (`RTC_DECAY = 0.97`), and injection from a small number of point vents per channel (`RTC_VENTS_PER_CHANNEL = 5`, `RTC_VENT_SIGMA = 2.2`, `RTC_INNOVATION = 0.65`). These rules are simulator code, not a trained model: the world dynamics never change in response to agent behavior or to any loss.

An agent does not read the field directly. Perception passes through a no-oracle seam: agents receive only lagged, windowed, noisy local sensor readings (`RTC_SENSOR_WIN = 5`, `RTC_SENSOR_SIGMA = 0.8`, `RTC_SENSOR_LAG = 3`) plus their own body state and any decoded messages — never the ground-truth patch value, edibility, or any founder identity used as a policy input. World truth is fenced to the eating/consequence resolver. We are precise about what the speaker conditions on when it chooses a message: the input to each candidate post (via `rtc_g1_run`) is a **lagged, noisy FULL_BASIS sensor read** — i.e. *not* pure body-local private perception, but also *not* an edibility/world-truth oracle (no ground truth enters any agent decision). This privileged-but-non-oracle input is precisely why we keep our claims at "**decodable signalling code**" rather than "grounded": the channel can be decoded and is behaviourally useful, but the speaker's richer-than-local input means we do not assert that its symbols track world state per se.

Survival is metabolic (`rtc_metabolism.py`). Each agent carries an energy reserve (`RTC_ENERGY_INIT = 50.0`) and pays a fixed upkeep every step (`RTC_UPKEEP = 0.18`). When it eats, the edible value of a patch is a dot product between the local 4-channel patch values and a fixed per-agent metabolic weight vector seeded from `N(0,1)`; the same patch is nutritious to one metabolism and useless or toxic to another. A value above an eat threshold (`RTC_EAT_THETA = 0.18`) yields clipped energy gain (`RTC_EAT_CLIP = 3.5`); a sub-threshold value costs energy (`RTC_TOXIN_COST = 0.7`); a value below the toxic floor is immediately lethal. The lethal threshold used by every banked g1f-arc run is `RTC_TOXIC_DEATH = -0.9`, set by each g1f harness via `os.environ.setdefault` and recorded in the verdict provenance; the unset `config/prereg_rtc.py` default is `-1.8`, but every g1f runner overrides it to `-0.9`, so `-0.9` is the value the entire results arc rode on (see Appendix). An agent whose energy reaches zero dies. Death is therefore a hard, hand-coded consequence of foraging in a flowing field — the selection pressure the rest of the study rides on.

### 3.2 The agent and its code-table

The agent body is intentionally trivial so that communication is the only thing under evolutionary pressure. The communicating object is a single `UnifiedIO` tied code-table of shape K × BANDS × VOCAB = 4 × 5 × 6 (`RTC_K = 4`, `RTC_BANDS = 5`, `RTC_VOCAB = 6`), initialized from `N(0,1)`. The table is *tied*: the same parameters drive both speaking and listening, so a mutation that changes how an agent emits also changes how it decodes. A speaker emits by taking, per channel, the `argmax` over the VOCAB symbols; a listener predicts by taking, per channel, the `argmax` over the BANDS value-bins. This coupling is load-bearing for the later convergence argument: emit and predict cannot drift apart independently.

We quantify communication with mutual intelligibility (MII): a set of referents is probed through one agent's `emit` into another agent's `decode`, scoring exact recovery of the full tuple. The referent space is the band×channel product, so chance recovery is 1 / N_REFERENTS = 1 / 5⁴ = 1/625 ≈ 0.0016. Within-founder (WF) and cross-founder (CF) MII are read off the off-diagonal of the full population×population MII matrix, filtered by lineage.

### 3.3 Reproduction, and the from-scratch invariants

Reproduction is mutation plus selection, with no gradient on the survival task. Offspring are produced by adding Gaussian noise to the parent's tied code-table (the kin arm preserves the founder lineage label across reproduction). Selection has two pre-registered variants used across the arc: hard top-50% truncation, and a soft tournament (k = 2) on lineage-shared fitness for niching. A population evolves over 28 generations for the headline co-evolution, kin-only diagnostic, and C1 arms; the later cross-lineage pressure probes (C2X, C2X2, C2X3) run longer (40–48 generations, stated per stage in the Appendix). The timescale is therefore scoped per stage rather than blanket-28 throughout.

The from-scratch invariants are strict and stated here once for the whole study. (i) There is *no gradient on the survival task*: foraging and death drive only mutation-and-selection. (ii) The *only* gradient mechanism anywhere in the codebase is an optional cultural-imitation (distillation) helper, and it is *unused in every load-bearing arm* reported here. (iii) There are *no borrowed or pretrained weights and no language model*; agents start from `N(0,1)` random tables. PyTorch appears solely as a tensor library, not as a source of learned structure. Any code that arises is therefore grown-from-scratch within this world, under survival selection, from random initialization — the precondition for the co-evolution claims that follow.

## 4. Methods: The Attack-Machine

Every claim in this paper is the output not of a single experiment but of a procedure designed to *try to make us wrong before a reviewer can*. We call this procedure the **self-audit**: six interlocking disciplines that together turn an emergent-communication study from an anecdote into an auditable evidence object. We present them here as a reusable practice, separable from our particular world. Its single methodological claim is **bidirectionality** — a self-audit should catch *deflation* as readily as inflation — which we demonstrate with two in-scope catches reproducible from this paper's own artifacts: a pilot/formal false *negative* (§5.1) and an RNG-offset bug in the headline random-fitness control (§4.2, §5.1) that, once fixed, slightly strengthened the result.

### 4.1 The mutual-intelligibility (MII) metric

The unit of evidence is not weight similarity or message entropy but **inter-agent codebook agreement**: can one agent decode what another emits? For an ordered pair of agents, we sample a fixed set of referents, route each through the speaker's `emit` and the listener's `predict`, and score **exact-tuple recovery** (the decoded tuple must match the referent exactly). With the code-table tied at K×BANDS×VOCAB = 4×5×6 and a referent set of 625, chance is 1/625 ≈ 0.0016. This unforgiving, no-partial-credit operationalization means an above-chance MII reflects a reconstructed *symbol-to-referent mapping*, not two networks that happen to look alike. MII quantifies agreement between two agents' codebooks, **not** world-state reference. Whether the symbols are *grounded* in world state per se is a separate property; in-scope we show only that the channel is decodable and behaviourally useful (the gate-0 content test below; world-state grounding is examined in a companion probe, g1d, cited once in §7.1 as supporting and otherwise out of scope). We also do not test compositionality, productivity, or displacement; those richer "language" properties are untested here, so we use "signalling code" / "shared convention" outside the Discussion accordingly.

We compute MII at two scopes. **Within-founder MII (WF)** pairs agents from the same founder lineage; **cross-founder MII (CF)** pairs agents from distinct founder lineages coexisting at the same generation. The WF/CF split is what lets us distinguish a *private kin code* (WF high, CF at chance) from a *public code* (CF also above chance). Throughout this arc WF runs ≈0.16 (~100× chance) while CF sits at the floor (e.g. diagnostic CF 0.00086; C1 soft96 CF 0.00153 ≈ floor 0.0016) — the picture that anchors the "decodable but private" conclusion.

To guard against MII being a measurement artifact, every stage runs two pre-logged sentinels. A **lineage-shuffle** control permutes lineage labels: if the metric truly keys on lineage, CF should spike under shuffling — it does (CF 0.0009 → 0.065 in the diagnostic; 15–23× across C1 cells). A **gate-0 content test** compares open messaging against muted and scrambled channels: open ≫ mute/scramble with CIs excluding zero (diagnostic open 0.31 vs mute 0.03 / scramble 0.05) establishes that message *content* is load-bearing, so a floor-level CF means "non-kin cannot decode," not "messages are irrelevant."

### 4.2 The architecture-matched random-fitness control

A selection-for-communication result is only interpretable against a control that differs in *exactly one thing*. Our control is **not** communication-ablated: **both arms carry the full `UnifiedIO` code-table**, the same initialization, and the identical evolutionary loop and configuration. The **only** manipulated variable is the **reproduction-fitness signal** — survival in the treatment arm versus uniform-random in the control — with the reproduction RNG **byte-matched between arms** (an independent generator supplies the fake fitness, offset from the shared stream). We therefore call it the **random-fitness control**. Because everything but the fitness signal is held fixed, an MII margin over this control cannot be attributed to extra capacity, a different population schedule, or a luckier seed set.

Crucially, this control isolates *selection for communication* from **clonal descent-convergence**: the random-fitness arm **also** fixates to ~one lineage, so any heritable code-table drifts toward self-similarity simply because survivors share ancestry. Its MII of **0.1174** is therefore the tied-architecture-plus-clonal-descent baseline (~73× the 1/625 floor), not zero. At gen 28 the survival arm reaches MII **0.17219**, a **paired** win-margin (per-seed survival-minus-commblind difference) of **+0.0548, 95% CI [0.0234, 0.0849]**, both arms formal and paired same-init at n=48. Selection *sharpens* a largely descent-determined channel; it does not originate it (the full decomposition and its transient decay are in §5.1).

This control also carries the project's two hardest lessons, both caught by the self-audit (§6). First, a *formal* treatment was once compared against a *pilot* control, manufacturing a false **negative** — encoded here as the rule **a control is valid only if it is byte-for-byte the same regime as its treatment**. Second, a multi-model adversarial review found that an earlier version of this very control drew its fake fitness from the *shared* reproduction RNG, partially offsetting the two arms; fixing it (an independent RNG for the fake fitness) and re-running at n=48 left the conclusion intact and slightly *strengthened* the margin (+0.045 → +0.0548). Both episodes are deflation-side catches.

### 4.3 Multi-seed bootstrap-CI win-margins

We report no single-seed positives. Every load-bearing comparison runs **≥6 seeds** (the headline uses 48), and the quantity reported is a **win-margin with a bootstrap confidence interval over per-seed paired differences**, not a p-value against an arbitrary null. The pairing is essential and we compute it per-seed: the headline win-margin is the mean over seeds of (survival_MII[s] − control_MII[s]), and the cross-lineage margins are likewise the mean over seeds of (CF_open[s] − baseline_max[s]). This per-seed pairing matters because a naive global-mean computation can spuriously flip a cross-lineage margin's sign positive (a Jensen-inequality artifact of taking the max before averaging); we therefore always report the paired quantity. A claim counts as supported only when the paired margin's CI excludes zero in the predicted direction; a CI that includes zero is reported as such and never upgraded. The headline check is the random-fitness control of §4.2: the survival arm beats it by the paired win-margin **+0.0548, 95% CI [0.0234, 0.0849]** at n=48 (§5.1).

**Multiple comparisons.** Each stage has a single, pre-registered **primary endpoint** locked before the run: the headline stage's primary is the paired survival-vs-random-fitness MII win-margin; the kin-only diagnostic's primary is the CF-vs-WF comparison under the frozen coexistence gate; each C1 cell's and each cross-lineage probe's primary is the paired CF-margin-vs-floor. The sentinel checks (lineage-shuffle, gate-0 content, founder-quality control) are **secondary corroborating diagnostics**, not independent tests of the headline. We apply **no family-wise correction**, by design: there is a single locked primary per stage, so there is no within-stage multiplicity to correct. The "pressure sweep" of §5.4 is **not** a single pre-registered multi-arm experiment but a *post-hoc synthesis* of three separately pre-registered probes (C2X, C2X2, C2X3); we treat it as triangulation, not as a corrected family.

### 4.4 Frozen pre-registration

Each stage's specification — its hypotheses, its coexistence gates, its outcome labels, and its sample size — is **locked in a versioned spec before the run** (e.g. the kin-only diagnostic spec v4.1 at commit 67ce59b; the C1 probe spec v5.1). Gates and labels are not moved after seeing data. The discipline's value is most visible when it costs us something: the diagnostic's frozen ≥2-coexisting-generations gate was met by only 5/16 seeds, forcing the pre-registered verdict `INCONCLUSIVE_NO_WINDOW` even though the *direction* (private codes) was unanimous and strong — we banked the honest inconclusive rather than relabel the gate.

### 4.5 The no-oracle redline

The central failure mode in this literature is leakage: an agent that secretly reads world truth can fake communication. We draw a hard **no-oracle redline** — agent decisions may read **only decoded messages, own body state, and legal sensors**, *never* true `world.patch`, edibility, target-code, or founder-id-as-policy-input. World truth is fenced to the eat/consequence resolver and to explicit oracle *controls* (used to measure ceilings, never to act). A CI static-analysis scan (`_redline_scan`) enforces a subset on the core runner (`rtc_g1f_coevolve.py`): it fails the build if `world.patch` is read outside `_run_episode` or if any of three banned training calls (`cotrain_factored`, `train_relay_channel`, `find_nearest`) appears. The remaining conditions — no edibility, target-code, or founder-id as a policy input, plus the cross-lineage runners' forced-listening logic — are enforced by construction and audited by code-grounded review (§4.6); the automated scan does not yet cover the C2X/C2X2/C2X3 runners.

### 4.6 Code-grounded adversarial multi-agent review

Before any spec is locked, it is attacked by independent reviewers reading the *actual runner code*, not a prose summary — code-grounded adversarial review. This caught, e.g., the C2X "content-free" auto-label as a viability-crash mislabel, and forced two reviewer contradictions in C1 (the fitness-sharing divisor and the window gate) to be adjudicated on the record before locking.

### 4.7 The tier rule and the provenance triple

Two governance rules bind the above. The **tier rule (frozen):** a run at **n ≤ 8 may only conclude POSITIVE or INCONCLUSIVE — never a scoped negative; a negative is banked only at n = 16.** Accordingly, the cross-lineage CF-at-floor result is reported as "a public code did not bootstrap under the regimes tested" *with a stated mechanism, across seeds and across three heterogeneous interventions*, but explicitly **not as a formal negative** — at n = 8 it remains on the positive-or-inconclusive tier, and we say so in-text in the same sentence each time.

Finally, every result self-describes via a **provenance triple**: the immutable **verdict JSON @ git commit** (with embedded `command_hint`, env, `effective_config`, `git_commit`, `git_dirty` via `make_provenance`), the **CLAIM_LEDGER** mapping each claim to its `verdict-file @ commit`, and the **RUNBOOK** giving the exact regeneration command per stage. Re-runs are seeded and deterministic — same command at the same commit yields a byte-identical verdict, overwriting only the working file while git preserves the banked version. One caveat: the g1f *headline* predates the provenance-block discipline, so the headline numbers come solely from re-running the n=48 RNG-fixed random-fitness control (`rtc_g1f_commblind_verdict_formal48_rngfix.json`); the legacy `reconciled`/`power_analysis` artifacts hold superseded pre-rngfix values and are not relied on (see Appendix and RUNBOOK).

## 5. Results

### 5.1 Co-evolution is real, but kin-scoped

The first question is the existence question: does a decodable communication channel co-evolve from random weights under survival selection alone? In the g1f configuration of the lethal-vent world, the answer is yes — but the scope of that "yes" is the load-bearing part of the result, and we state it in the same breath as the claim.

**The headline (a decomposition).** Under formal settings (population reproduced by survival fitness, no gradient on the task, no oracle labels, no borrowed weights), the co-evolved channel reaches a mean cross-agent mutual intelligibility (MII) of **0.17219**, against an architecture- and process-matched **random-fitness control** at **0.1174**. The control is identical in every respect — same `UnifiedIO` code-table, same initialization, same mutation and reproduction machinery, same number of generations — *except* that reproduction is driven by a uniform-random fitness signal rather than survival, with the reproduction RNG byte-matched between arms (an independent generator supplies the fake fitness). This isolates *selection for communication* from the confound of **clonal descent-convergence**: the random-fitness arm **also** fixates to ~one founder lineage, so its MII is not zero but the descent/architecture baseline that any heritable code-table reaches simply because survivors share ancestry. That baseline is already **0.1174 — ~73× the 1/625 ≈ 0.0016 chance floor**. At the pre-registered gen-28 fixation point, survival selection then adds a **paired** bootstrap win-margin (the per-seed survival-minus-commblind difference) of **+0.0548, 95% CI [0.0234, 0.0849]** (n = 48; the interval is recorded as `survival_minus_commblind_ci` in `rtc_g1f_commblind_verdict_formal48_rngfix.json` — the RNG-fixed control — which also holds the per-arm means survival 0.17219 and random-fitness 0.1174). This rngfix verdict is the **sole** source for the headline win-margin, CI, and control mean; the new interval is *not* cross-recorded anywhere else. The older derived artifacts `rtc_g1f_reconciled_verdict.json` and `rtc_g1f_power_analysis.json` predate the RNG-fix and hold the *superseded* pre-rngfix values (win-margin +0.045, 95% CI [0.02017, 0.06982], random-fitness control 0.1271); they are legacy, have not been regenerated against the rngfix control, and the paper does not draw any headline number from them. The interval excludes zero. The margin is modest in absolute terms; at the n=8–16 used by the later stages it would be underpowered (≈0.4–0.5 power), but at n=48 it is well resolved. The reading is therefore a decomposition rather than an origination: the tied architecture plus clonal descent reaches MII 0.1174 on its own, and survival selection **sharpens and tunes** that channel by +0.0548 — selection raises mutual intelligibility above what shared ancestry alone produces, but it is not conjuring a channel from nothing.

**The MII is semantic, not a weight-similarity artifact.** MII is measured behaviourally — a referent is pushed through one agent's `emit` and another's `predict`, and we score exact-tuple recovery (chance ≈ 1/625 ≈ 0.0016, the frozen-null floor; to keep the two multiples distinct, the random-fitness descent baseline MII 0.1174 is ~73× this floor, while the within-lineage WF ≈ 0.16 is ~100× it). Two checks guard against the obvious objections. First, the random-fitness control above differs from the treatment only in the fitness signal, and the survival arm beats it on the *behavioural* metric, not on parameter overlap. Second, the channel's content is load-bearing: the pre-registered gate-0 sentinels confirm that open communication beats both the muted and the scrambled-message conditions on survival (`open_beats_mute_survival` and `open_beats_scramble_survival` both pass — a deliberately limited use of the legacy reconciled verdict's boolean sentinel flags only, not its superseded pre-rngfix win-margin), so open-channel content is behaviourally used by the agents to survive, rather than incidental structure that survives muting. This establishes the channel is decodable *and* behaviourally useful; whether its symbols additionally track world state per se is a separate result (g1d, out of scope here). The no-oracle redline holds throughout — agents never read truth labels — and no pretrained or borrowed weights enter the system at any point.

**The scope (load-bearing).** The survival arm fixates to a single founder lineage: across n=48 seeds the mean dominant-lineage share is **99.1%** (95% CI [97.3%, 100%]), the inverse-Simpson effective number of lineages **N_eff = 1.02**, and **47 of 48** seeds end essentially single-lineage (`rtc_g1f_lineage_share_verdict.json`). The co-evolved channel is therefore **kin-lineage decodability — a private kin code, not a public cross-lineage shared convention**. Everything in §5.2–§5.4 follows from drawing this line precisely: the agents that decode each other are kin, and we have not yet shown that strangers can. We claim the channel grows; we do not claim it is shared. This is scoped to *this world, this architecture, and this 28-generation timescale*; whether the same holds in richer worlds is untested and is the subject of the discussion.

**Robustness: the margin is a transient acceleration (secondary, post-hoc).** The +0.0548 above is the **pre-registered gen-28 primary** endpoint, measured at the fixation point, and it stands at that timescale. To ask whether it *persists*, we ran a secondary, post-hoc transient probe (n=24, checkpoints at generations 28/56/100/150; `rtc_g1f_transient_probe` @ddc2822) — explicitly not a pre-registered primary, so it is tied to the multiple-comparisons paragraph (§4.3) as a corroborating diagnostic, not an independent test. The paired margin **decays monotonically**: gen 28 survival 0.17415 / random 0.11725 / margin **+0.0569, CI [0.0162, 0.1021]** (excludes 0); gen 56 **+0.0489, CI [0.0151, 0.0821]** (excludes 0); gen 100 **+0.0329, CI [−0.0014, 0.0687]** (includes 0); gen 150 **+0.0249, CI [−0.0072, 0.0571]** (includes 0). The mechanism is visible in the two arms separately: the survival-arm MII stays roughly stable (~0.16–0.17 across all four checkpoints) while the **random-fitness control's MII *rises*** (0.11725 → 0.13805) as single-lineage clonal fixation alone keeps accumulating cross-agent decodability over generations — so the gap closes from below. The reading is therefore (a) that survival selection produces a **transient acceleration** of clonally-shared decodability rather than a sustained equilibrium difference — by ~gen 100 the control's clonal fixation reaches comparable MII — and (b) that this is the empirical justification for scoping the headline to 28 generations. It reinforces, rather than refutes, the §5.1 decomposition: within-kin decodability is fundamentally a **clonal-descent** phenomenon (ties to the 0.1174 descent baseline) that survival selection merely *accelerates*. We keep the gen-28 positive as the headline; the decay narrows the *temporal* scope of the claim, it does not overturn the positive.

**On the baseline value (one realization).** The random-fitness control's MII (0.1174) is one stochastic realization that depends on the (arbitrary) control-RNG seed; it is not a unique constant. An independent realization yields a comparable-or-larger margin (≈+0.065), so the *positive* margin is robust to that choice even though the exact baseline value is not.

**Methodology hook (the audit is bidirectional).** This positive result has twice been corrected in the *deflation* direction. It first began life as a banked *negative*: an earlier "g1f clean negative" reported no co-evolution — but it was a false negative the authors had introduced, a *formal* co-evolve arm compared against a *pilot* control (a configuration mismatch in which the control ran with different settings than the treatment). The self-audit, prompted by the direct question "did we kill a real effect for the paper's sake?", caught the mismatch; the load-bearing knob was generation count (lifting it from 7 to 28 let the effect rise to its plateau around generation 12–14, and the pilot had truncated mid-rise). Later, a multi-model adversarial review found that the random-fitness control had drawn its fake fitness from the *shared* reproduction RNG, partially offsetting the two arms; fixing it (independent RNG) and re-running at n=48 left the conclusion intact and slightly *strengthened* the margin (+0.045 → +0.0548). The same discipline that prevents false positives — same-config treatment-versus-control — is exactly the discipline that exposed both deflations; §6 develops bidirectionality as the methodological claim.

### 5.2 Why kin-only? (diagnostic)

Section 5.1 established that, within g1f, a decodable channel co-evolves under survival selection but does so inside a population that fixates to a single founder lineage (dominant-lineage share 99.1%, N_eff 1.02). That leaves two competing explanations for its kin-scoped character, and they have opposite implications. Under hypothesis **A (collapse-limited)**, non-kin lineages *would* be mutually intelligible if they coexisted long enough — the kin-only signature is an artifact of fast founder collapse, and is in principle fixable simply by maintaining diversity. Under hypothesis **C (private codes)**, coexisting non-kin lineages diverge into mutually undecodable private codes, so the channel is intrinsically private and a public code would require shared grounding rather than mere coexistence. To discriminate, we ask: at generations where at least two founder lineages coexist, is cross-founder mutual intelligibility (CF) near the within-founder ceiling (WF → A) or down at the architecture floor (CF ≈ chance → C)?

We pre-registered this comparison under a frozen gate (`STAGE_KINONLY_DIAGNOSTIC_SPEC.md` v4.1, commit 67ce59b): a seed contributes only if it shows a "window" of at least two generations in which at least two lineages each keep ≥3 living agents, and the verdict is declared `INCONCLUSIVE` unless at least 8 of 16 seeds clear that bar. The frozen verdict is **`INCONCLUSIVE_NO_WINDOW`**: only **5/16** seeds opened a window, because founder collapse is too fast to satisfy the coexistence requirement (`rtc_g1f_kinonly_diagnostic_verdict.json` @6aaefe2, n=16, formal). We report this as inconclusive rather than relabeling the gate after seeing the data — the gate held. This honesty is itself a methodology point (§6): the diagnostic could not be *concluded* at the pre-set power, and we say so in-text rather than promoting a directional signal to a verdict.

The **direction**, however, is strong and unanimous, and it points to **C**. Across the measurable seeds, within-founder MII is **0.159** (≈100× chance, where chance = 1/625 ≈ 0.0016) while cross-founder MII is **0.00086** (≈ chance, ≈ the frozen-mixed floor of 0.00163 [0.00146, 0.0018]); the ratio CF/WF is **0.5%**, and all **5/5** measurable seeds (the 5 of 16 that opened a coexistence window) take the same form — class C in the verdict's `per_seed_class_counts`. So whenever two non-kin lineages did coexist, they decoded each other essentially at chance while each decoded its own kin ~100× above chance — diverged private codes, not a latent shared code awaiting coexistence.

Two pre-registered sentinels rule out the obvious artifacts. (i) **Lineage-shuffle:** permuting lineage labels lifts CF from 0.0009 to 0.065 — a ~75× jump — confirming the metric truly keys on lineage membership rather than reporting a generic floor. (ii) **Gate-0 (content load-bearing):** open-channel populations survive far better than muted or scrambled ones (fraction alive 0.31 versus 0.03 when muted and 0.05 when scrambled, with the open−mute CI [0.195, 0.367] and open−scramble CI [0.168, 0.344] both excluding zero). Message content is therefore load-bearing for survival; the floor-level CF is genuine cross-lineage unintelligibility, not an inert channel.

The result is scoped accordingly. Within g1f (this world, this architecture, a 28-generation timescale), the direction unanimously indicates private kin codes, but the diagnostic is **not** a concluded test — the frozen window gate was met by too few seeds. We do not yet claim that maintaining diversity *cannot* rescue cross-lineage intelligibility; the directional signal motivates a test in which diversity is actively sustained, which is the subject of §5.3.

### 5.3 Maintaining diversity is necessary but not sufficient

The diagnostic of §5.2 reached an honest *no-window* verdict — founder lineages collapsed before we could measure whether coexisting non-kin lineages understood each other — but pointed, unanimously and across every measurable seed, toward explanation **C** (private codes) over **A** (collapse-limited). This left a single clean hypothesis to attack: if we *prevent* the collapse and force non-kin lineages to coexist for many generations, will cross-lineage intelligibility appear (A: kin-only is an artifact of truncation) or will it stay at the architecture floor (C: kin-only is genuine private-code divergence)? Section 5.3 welds the collapse diagnosis directly into a minimal *maintain-diversity* intervention and asks the question.

**Design.** We ran a 2×2 factorial — selection {hard top-50% truncation; soft = tournament-k2 with lineage fitness-sharing, divisor = generation-start same-lineage count} × population {16, 96} — under identical configuration and seeds (pop16 n=16; pop96 n=8). Coexistence is scored by a *dual* gate (≥2 lineages each with ≥3 living agents **and** inverse-Simpson N_eff ≥ 2.0) so that "diversity" cannot be faked by raw population scale. Cross-founder MII (CF) is computed over the full vectorised population×population referent-decode matrix; the chance/architecture floor is 1/625 ≈ 0.0016. The specification was locked at v5.1 (commit f99ed6f) after two adversarial reviews — both initially returning DO_NOT_LOCK — whose two direct contradictions (over the fitness-sharing divisor and the window gate) were adjudicated, plus a confirm pass (`rtc_g1f_c1_collapse_probe_verdict.json` @21dc0f3).

**The 2×2.**

| cell | opens windows | final N_eff | cross-founder MII (CF) | CF−FLOOR (CI) | WF | CF/WF |
|---|---|---|---|---|---|---|
| hard16 | no (3/16) | 0.94 (collapsed) | 0.00081 | −0.00068 [−.00095, −.00052] | 0.174 | 0.5% |
| hard96 | yes (8/8) | 1.44 (still collapses) | 0.00176 | +0.00017 [−.00045, +.00092] | 0.175 | 1.0% |
| soft16 | no (2/16) | 0.72 (collapsed) | 0.00072 | −0.00074 [−.00117, −.00031] | 0.096 | 0.8% |
| soft96 | yes (8/8) | 4.10 (sustained) | 0.00153 | −0.00007 [−.00031, +.00021] | 0.160 | 1.0% |

**Mechanism: population opens windows, niching sustains them.** The binary window label ("POP_DRIFT") is too coarse, so we refine it with the pre-logged N_eff trajectory. At pop=16 the population collapses regardless of selection (final N_eff 0.94 hard, 0.72 soft; only 3/16 and 2/16 seeds ever open a window). At pop=96 windows open in all 8/8 seeds, but only soft selection *sustains* the diversity: hard96 still collapses to a final N_eff of 1.44, whereas soft96 ends at N_eff 4.10 in all 8 seeds, holding 15–25 coexisting generations each. Maintaining lineage diversity at this scale therefore requires niching, not merely a larger population.

**Consistent with C, against the strong form of A (directional, n=8).** Within soft96 — the one cell that actually delivers sustained multi-lineage coexistence (n=8, pop≥96, 28 generations) — cross-founder MII still sits at the floor: CF 0.00153 versus FLOOR 0.0016 ≈ 1/625, with the CF−FLOOR confidence interval **[−0.00031, +0.00021]** straddling zero, and CF/WF ≈ 1% against a within-founder MII of ≈0.16 (~100× chance). Preventing collapse and sustaining 20+ generations of genuine non-kin coexistence did **not** lift CF off the floor. We read this as *consistent with* genuine private-code divergence (C) and *against the strong form of* the collapse-limited hypothesis (A) — but we report it as **directional at n=8, not a banked refutation of A**: the soft96 CF−FLOOR CI straddles zero, so this is a no-detectable-effect result at this power, and a banked negative would want n=16 (owed). Maintaining diversity is **necessary** — without it (pop16, hard96) coexisting lineages cannot even be measured — but, on this evidence, it is **not sufficient** for a shared, public code.

**Sentinels.** Three checks confirm the floor-level CF is a real divergence and not metric failure. (i) Lineage-shuffle: randomly relabelling lineages at coexisting generations raises CF 15–23×, so the metric truly keys on lineage identity rather than reporting a constant. (ii) Gate-0 content test: open-channel populations survive far above muted or scrambled ones (soft96 fraction alive: open 0.221 vs mute 0.056, scramble 0.061; open−mute CI [0.116, 0.220], open−scramble CI [0.112, 0.211], both excluding zero), so message content *is* load-bearing for survival — the chance-level CF is not "communication doesn't matter." (iii) Founder-quality control: the Spearman correlation between a founder's generation-0 fitness (and self-MII) and its final population share is low across all cells (0.03–0.20), so which lineage wins is drift / winner-take-all, not a lucky high-fitness start.

**On WF: not a co-evolved kin "ceiling."** A natural misreading would take the within-founder MII (WF ≈ 0.16) as a co-evolved *kin* code — an upper bound that cross-founder pairs fail to reach. We resist this. Different founders begin at ≈zero cross-founder intelligibility because their code-tables are independently seeded random draws, and they never converge; the WF ≈ 0.16 a clonal family shows is the **inherited self-intelligibility of descendants of one founder**, not a ceiling that two lineages *co-evolved* toward. We therefore do **not** cite WF as evidence of a co-evolved kin code. The substantive null rests instead on the *cross-founder* comparison: in soft96 (the sustained-diversity cell), CF stays at the floor through 20+ actively-sustained coexisting generations.

**Scope.** These conclusions are bounded to this world, this UnifiedIO architecture, and this 28-generation timescale. The collapse boundary itself is observed only at the cases tested: pop=16 is fatal regardless of selection (small-population drift) and niching sustains diversity at pop=96 — where exactly the boundary lies, and whether the pattern holds in other worlds or architectures, is untested beyond these cells. The directional C reading rests on n=8 for pop96 (unanimous 8/8, with a stated mechanism, but a CF−FLOOR CI straddling zero), which places it in the positive/inconclusive evidence tier; we do not bank it as a formal negative, which would want n=16.

### 5.4 The wall: no public code bootstraps

Section 5.3 left us with a sharpened question. Within-kin a decodable channel co-evolves, and even when lineage diversity is actively sustained (soft96, *n*=8), coexisting non-kin lineages still decode one another only at the architecture floor (directional at n=8). The obvious next move is to make survival itself *depend* on decoding non-kin: if cross-lineage intelligibility paid the rent, would a public code be selected up? We attacked this with **three heterogeneous interventions** (C2X, C2X2, C2X3) — *different designs and scaffolds, run for 40 vs 48 generations* — each LOCKED only after code-grounded adversarial review. They are **not** a single graded dose-response sweep and we do not present them as one; rather they **triangulate the same null** from three directions, each placing different cross-lineage decoding demands on survival. Across all three, a public cross-lineage code did not bootstrap under these regimes (n=8, positive-or-inconclusive tier).

**The three interventions.** The **viable cell (C2X2 stationary scaffold, 0.25 non-kin)** is the load-bearing one because the population stays alive: mean fraction alive 0.1574, clearing the pre-registered viability bar (≥0.15 absolute, ≥0.5× the kin-only C1 reference of 0.2722), with coexistence windows in all 8/8 seeds. Yet cross-founder MII (CF) sits at the floor — CF = 0.00156 against a chance/floor of 0.00159, a *negative* margin of −0.00065, 95% CI [−0.00095, −0.00038] (CI excludes zero) — while within-founder MII (WF) is 0.156. This is the cleanest single cell: viable population, sustained windows, and CF still pinned below the floor. (One caveat: the C2X2 verdict auto-labels both arms `CONTENT_FREE_SHORTCUT`; the ledger records this as a mislabel — the content gate is uninformative at this viability, not a degenerate channel.) The **forced-quota intervention (C2X3, 0.5 non-kin, per-generation logged)** *corroborates with a confound*: it imposes a K=4 diversity quota plus constant non-kin pressure, but the quota **did not actually hold diversity even at zero non-kin pressure** (its QUOTA_KINONLY control reaches inverse-Simpson N_eff median 2.653, below the pre-set 3.0 diversity bar), so we treat C2X3 as corroborating rather than as the place the mechanism becomes "visible." With the quota in place the population still goes boom-bust: mean alive 0.0998, below the viability bar, and CF never leaves the floor (per-generation CF max 0.00579 across all seeds and generations, versus chance ≈ 0.00159; seed-aggregate CF = 0.00192, margin −0.00086, CI [−0.00139, −0.00032]). The **cold 100%-non-kin intervention (C2X)** shows the viability failure mode: mean alive collapses to 0.0455 versus the kin-only C1 reference of 0.2718 (the C1 reference reads slightly differently per probe — 0.2722 in C2X2, 0.2718 in C2X — because each re-measures its own kin-only arm) and the population dies before selection can act on communication at all; CF pins to the floor (CF_OPEN = 0.00166, margin −0.00047, **CI [−0.00097, 0.00013] — which includes zero**, i.e. consistent with chance, unlike the 0.25 and 0.5 cells whose CIs exclude zero).

**Headline.** Across all three interventions, *cross-founder MII never rose above chance.* Turning the cross-lineage screw did not buy a public code; where it bit at all, it crashed survival. We do not claim a graded dose-response: the three probes differ in design, scaffold, and generation count (40 vs 48), and we report only the qualitative pattern that CF stayed at chance under every regime we tried, with the viable 0.25 cell and the sustained-diversity C1 soft96 cell carrying the substantive weight.

**Mechanism.** The pressure-vs-survival reading rests primarily on the **viable 0.25 cell** (alive, diverse, yet CF at floor) together with **C1 soft96** (sustained diversity, CF at floor). Non-kin codes are mutually undecodable (§5.3), so an agent forced to listen to non-kin signals receives, in effect, noise — and acting on noise costs foraging time. The C2X3 per-generation logs (read as corroborating, given the quota confound) show viable-and-diverse coexistence appearing only in transient bursts — per-generation fraction alive ranges from 0 to 0.7604 within single runs, spiking when diversity is high and collapsing immediately after — never holding long enough for cross-lineage selection to accumulate. This is a **pressure-vs-survival tension**: the very pressure needed to reward a public code starves the foraging that keeps a diverse population alive long enough to evolve one. A viable, diverse, *converging* regime is never reached in g1f under any intervention tried.

**Power tier and what this null is (and is not).** All arms here are *n*=8 — the POSITIVE-or-INCONCLUSIVE tier of our method. We therefore do not bank a FORMAL negative (that would require *n*=16). What we do report is that **a public code did not bootstrap under the regimes tested**: CF stays at the architecture floor across all seeds *and* across all three heterogeneous interventions, with a stated mechanism (forced non-kin listening starves foraging). This triangulation across heterogeneous probes is qualitatively stronger than an isolated underpowered null — but, per our calibration discipline, we claim only that under these regimes a public code *did not* bootstrap (positive-or-inconclusive tier; not banked as a formal negative), never that this architecture *cannot* produce one. The scope is explicit: this world (the RTC lethal-vent world, `rtc_world.py`), this architecture (the tied `UnifiedIO` code-table), this timescale (the cross-lineage probes run 40–48 generations; see Appendix).

**A methodology hook.** This section also exhibits the self-audit correcting itself. C2X's automatic verdict label was `CONTENT_FREE_SHORTCUT` — the rule fired on a floor-level content gate and concluded the channel had degenerated to a contentless shortcut. That label was a **mislabel**: at 0.0455 fraction alive the content gate is simply uninformative, and the real phenomenon is a viability crash, not a shortcut. The error was caught and corrected by examining the pre-logged continuous metrics (per-generation alive and CF trajectories) rather than trusting the coarse categorical auto-label — the same continuous-metric discipline that, in C2X3, turned an over-broad `DESIGN_FAILURE_VIABILITY_CRASH` flag into a legible mechanism.

**What this resolves for g1f (scoped).** For g1f, the cross-lineage chapter has a clear reading under the regimes tested: even when survival is made to depend on decoding non-kin, across three heterogeneous interventions and with diversity propped up by a quota (a quota that did not itself hold N_eff above the diversity bar), a public cross-lineage code did not bootstrap — the decodable code here remains kin-scoped. The natural reading is that crossing this wall needs a *richer world* (more niches and referents to sustain diversity under communication pressure, and shared referents worth a cross-lineage code for), which we take up in the Discussion as motivated future work rather than a deficiency of the present arc.

## 6. The Methodology, Demonstrated

The preceding sections used the self-audit to defend a set of positive and null claims. This section turns the lens around. The audit's distinctive value — its single methodological claim — is that it runs **bidirectionally**: it is built to catch its operators inflating a result, and it is equally built to catch them *deflating* one. We anchor this on **two in-scope, reproducible catches** from this paper's own artifacts: a pilot/formal false *negative*, and the RNG-offset bug in the headline random-fitness control. Two further in-scope episodes (holding a frozen gate to an honest inconclusive; adjudicating contradictory reviewers; reining in an over-coarse auto-label) show the discipline in routine operation. Bidirectionality is not an incidental property of the rig — it *is* the methodological claim, and we present the rig as a disciplined practice we recommend rather than a co-equal scientific contribution.

**It caught a false negative (§5.1).** The g1f line began with a committed "clean negative": survival selection did not appear to co-evolve a decodable channel. The self-audit exposed this as an artifact of the author's own setup — a *formal* co-evolution arm (28 generations) had been compared against a *pilot* control. The redline that demands a control share the *same configuration* as its treatment flagged the mismatch; the load-bearing knob was generations (the effect plateaus around generation 12–14, and the pilot had truncated mid-rise at 7). Re-run with both arms formal and paired at the same initialization, the survival arm beats the random-fitness control by an MII win-margin of **+0.0548, 95% CI [0.0234, 0.0849]** at n=48 (survival-arm MII 0.17219 versus the random-fitness control's 0.1174). A real, peer-review-relevant effect had been suppressed *for the paper's sake*; the audit recovered it. This is the direction conventional self-checking almost never covers.

**It caught an RNG-offset bug in the headline control (§4.2, §5.1).** A multi-model adversarial review (a Claude science panel, a Claude/Codex code bug-hunt, and a DeepSeek statistics audit) found that the random-fitness control drew its fake fitness from the *shared* reproduction RNG, so the two arms' reproduction draws were partially coupled and offsetting. Fixing it — an independent generator (offset +9e6) for the fake fitness — and re-running at n=48 left the conclusion intact and **slightly strengthened** the margin, from +0.045 to **+0.0548** (survival arm 0.17219, control 0.1174, CI [0.0234, 0.0849]). This is the second clean deflation-side catch: a bug whose correction made the headline *more* favourable, found precisely because the audit attacks the authors' own favourable result.

**It held a frozen gate to an honest inconclusive (§5.2).** The kin-only diagnostic pre-registered a coexistence gate (≥2 founder lineages coexisting). Only 5 of 16 seeds cleared it — founder collapse was too fast — so the pre-registered verdict returned `INCONCLUSIVE_NO_WINDOW`. The temptation here is to relabel: the *direction* was strong and unanimous (within-founder MII 0.159, ~100× chance, vs cross-founder MII 0.00086, ≈chance, across all 5/5 measurable seeds, every one class C). The frozen gate held; we report inconclusive on the window and direction-only on the lean, rather than promoting an underpowered signal to a result.

**It adjudicated two direct contradictions between independent reviewers (§5.3).** Locking the C1 collapse-probe spec required resolving two disagreements between independent code-grounded reviewers: the correct *fitness-sharing divisor* for lineage niching (resolved to the generation-start same-lineage count, so sharing is computed on a stable denominator) and the *coexistence-window gate* definition (resolved to a dual criterion — ≥2 lineages each with ≥3 living members *and* inverse-Simpson N_eff ≥ 2.0 — to de-confound diversity from raw population scale). Both reviews returned `DO_NOT_LOCK`; the spec was locked (v5.1) only after the contradictions were explicitly adjudicated and a confirm pass run. The machine is the forum in which reviewer disagreement is forced to a decision before any number is generated.

**It corrected an over-coarse auto-label using a pre-logged continuous metric (§5.4).** The C1 window classifier emitted the binary label `POP_DRIFT`, which undersells the mechanism. The pre-logged continuous N_eff trajectory refined it: population size *opens* windows (pop=16 collapses regardless of selection; pop=96 opens them), but only soft/niching selection *sustains* diversity (hard96 ends at N_eff 1.44 — still collapsing — while soft96 ends at N_eff 4.10, all 8 seeds, 15–25 coexisting generations). Logging a continuous metric *before* knowing the answer let us correct a label that a coarse gate alone would have shipped.

**The broader record.** Anecdotally and out of scope, the same discipline applied to a separate earlier line on a different substrate also caught inflation-side errors (underpowered pilots that did not survive powered re-runs); those artifacts are not part of this paper and we do not lean on them. The in-scope point stands on its own: a skeptical reviewer will note that the authors made errors — a false negative and an RNG-offset bug in their own headline control. That is exactly the point. The demonstration that the audit catches its own authors *in the deflation direction* — two reproducible in-scope catches, both of which, once corrected, supported rather than undercut the result — is the evidence that bidirectionality is a practice worth reusing.

## 7. Discussion

### 7.1 "Decodable but private to kin" and what it means for language-origin theory

The central finding of this arc is a dissociation that language-origin theory does not usually hold apart: a communication channel can be *decodable* without being *public*. Under survival selection alone, a decodable channel co-evolves from random weights and beats an architecture-matched random-fitness control (survival-arm MII 0.17219 vs control 0.1174; paired win-margin +0.0548, 95% CI [0.0234, 0.0849], n=48), with the caveat — central to our reading — that the control already reaches 0.1174 (~73× chance) from clonal descent alone, so selection *sharpens* rather than originates the channel, and that this gen-28 margin is a transient acceleration the control's own clonal fixation closes by ~gen 100 (§5.1). The channel is semantic, not weight similarity — within-founder mutual intelligibility (WF) sits near 0.16, roughly 100× the 1/625 ≈ 0.0016 chance floor, and the content is load-bearing (the gate-0 sentinel has open ≫ mute/scramble, CIs excluding zero). A companion probe (g1d, out of scope) additionally finds the channel's symbols track world state, supporting a *grounding* reading; but our in-scope MII measures inter-agent codebook agreement, not world-state reference, and we keep the two claims separate. Across every regime we tested, cross-founder MII (CF) never rose above chance (diagnostic: 0.00086 vs 0.159 within-founder; CF/WF ≈ 0.5%). The code is real and it means something — but only to kin.

For theories of language origin, this matters because intelligibility is often treated as the hard part and publicness as a near-automatic consequence: once signals are decodable, a shared code is assumed to follow. Our minimal world separates the two. A decodable code emerged readily; publicness did not emerge at all. The reason is mechanistic rather than incidental: non-kin codes are mutually undecodable, and within a single lineage there is no selective pressure to be understood by strangers one will, in practice, almost never meet (the formal population fixates to ~99% a single founder lineage, N_eff 1.02). A private kin code is, in this world, an *attractor* — the path of least resistance under survival selection.

**What is non-obvious here, relative to kin-selection and iterated-learning theory.** Kin selection makes a *kin-private* outcome unsurprising once relatedness is high; iterated-learning theory predicts that *shareable* structure needs a transmission bottleneck. Two things in our result are not handed to us by either theory. First, a *decodable* channel emerged **without being trained or designed in** — no gradient on the channel, no oracle, no imposed bottleneck — so the intelligibility is grown-from-scratch under survival selection, not installed. Second, the privacy we report is shown as **"founders never converge"** — independently seeded lineages starting at ≈zero cross-founder intelligibility and staying there through 20+ actively-sustained coexisting generations (C1 soft96) — rather than as a mere collapse to a single lineage. That a *sustained-diversity* regime still yields no convergence is the non-trivial claim, distinct from the trivial observation that a collapsed population is monoclonal.

### 7.2 Why a public code plausibly needs more world, not more selection

The natural rescue is to *select for* cross-lineage understanding — make survival depend on decoding non-kin. We did exactly this across three heterogeneous interventions (different designs and scaffolds, 40 vs 48 generations) that triangulate the same null rather than forming a graded dose-response. It did not work, and the way it failed is informative. In the viable 0.25 cell the population stayed alive (alive ≈ 0.16) but cross-founder MII held at floor (CF margin −0.00065, CI excluding zero); under the 0.5 forced-quota intervention survival became boom-bust (mean alive 0.10, range 0–0.76) with CF never exceeding 0.0058 across all seeds and generations (with the caveat that its diversity quota did not actually hold N_eff above the 3.0 bar); under cold 100% non-kin the population simply crashed (alive 0.045 vs 0.272, CF margin CI including zero, consistent with chance). Where the pressure bit at all, it only crashed survival harder. We call this the **pressure-vs-survival tension**: forcing agents to listen to non-kin whose codes they cannot yet decode degrades foraging and kills them before selection can act on communication, so a regime that is simultaneously *viable, diverse, and converging* is never reached for long enough to matter.

This points away from "more selection pressure" and toward "more world." Two ingredients appear to be missing. First, **shared referents worth naming**: in a world thin enough that a private dialect suffices for survival, there is little fitness reason for codes to align across lineages. Second, **sustainable diversity under communication pressure**: a public code presupposes multiple lineages that both persist *and* benefit from understanding one another. We showed that maintaining diversity is necessary but not sufficient — niching (tournament selection plus lineage fitness-sharing) sustained genuine multi-lineage coexistence at pop=96 (soft96 final N_eff 4.10, 8/8 seeds, 15–25 coexisting generations), yet cross-founder MII still sat at the floor (CF 0.00153 ≈ floor 0.0016; CF−FLOOR CI [−0.00031, +0.00021] includes zero, so directional at n=8) while within-founder MII held near 0.16. Preventing collapse does not, on this evidence, manufacture a common tongue. Whether the binding constraint is the world/ecology or the optimizer is the question we take to future work (§8): a richer world that sustains diversity *and* makes cross-lineage understanding pay, and/or an oracle-clean, consequence-driven learner that could re-couple drifting lineages, are the two candidate next ingredients — motivation for, not a result of, this arc.

### 7.3 Relation to the iterated-learning bottleneck

We deliberately did *not* impose a transmission bottleneck in the sense of iterated-learning models (Kirby, Cornish & Smith, 2008), where a learnability constraint at each generational hand-off pressures codes toward shared, compressible structure. Our agents inherit weights by mutation and selection, not by learning from a teacher through a narrow channel. The pressure-vs-survival tension we observe can be read as suggesting *why* such a bottleneck — or an ecological analogue of one — may be a prerequisite for a public code: without a mechanism that repeatedly forces codes through a shared, learnable interface, lineages drift into mutually private dialects and there is no force pulling them back together. We frame this as a hypothesis the arc *motivates*, not a result it establishes. We did not run an iterated-learning condition, so we cannot claim a bottleneck would succeed where survival pressure did not; we can only say that, absent any such mechanism, no public code arose across the regimes we tested.

### 7.4 Honest limits

These claims are scoped narrowly and on purpose. They hold for **one hand-coded world** (the RTC lethal-vent substrate), **one architecture** (the tied UnifiedIO code-table, random N(0,1) init), and **per-stage timescales** (28 generations for the headline co-evolution, kin-only diagnostic, and C1 arms, where the channel effect plateaus around generation 12–14; 40–48 generations for the cross-lineage pressure probes). The headline co-evolution result is well powered (n=48), though its absolute margin is modest and decomposes into a clonal-descent baseline (MII 0.1174) plus a selection increment (+0.0548) — and a secondary, post-hoc transient probe (n=24) shows that increment is a **transient acceleration** that decays as the random-fitness control's own clonal fixation catches up (margin CI includes zero by ~gen 100), which is why we scope the headline to a 28-generation timescale rather than claiming a sustained equilibrium difference. The cross-lineage outcome is *not* a formal negative: a public code "did not bootstrap under the regimes tested (n=8, positive-or-inconclusive tier), across three heterogeneous interventions, with a stated mechanism." Banking a formal negative would want n=16, and we explicitly decline to do so here. Accordingly we make no impossibility claim — under these regimes a public code *did not* bootstrap, never that it *cannot*. Nor do we claim the failure lies in "the bottleneck rather than the model or optimizer": we have no positive control in which a public code does emerge, so we cannot attribute the null to any one cause. What we offer is a legible mechanism in a minimal world — a decodable code grows kin-privately, and the tension between communication pressure and survival marks where, and plausibly why, a public code does not yet follow.

## 8. Conclusion and Future Work

This paper makes one coupled claim with two faces. **Scientifically**, in a minimal hand-coded 2-D survival world, with no gradient on the task, no oracle or truth labels, and no borrowed or pretrained weights, a *decodable* communication channel **co-evolves from random weights under survival selection alone** — but the result decomposes. The architecture-matched **random-fitness control**, which manipulates only the reproduction-fitness signal and also fixates to ~one lineage, already reaches MII **0.1174** (~73× the 1/625 ≈ 0.0016 chance floor) from clonal descent alone; at the pre-registered gen-28 fixation point survival selection then **sharpens** this channel by a paired MII win-margin of **+0.0548 (95% CI [0.0234, 0.0849], n=48; survival-arm MII 0.17219 vs control 0.1174)** — a margin a secondary post-hoc probe (n=24) shows is a **transient acceleration** that decays to a CI including zero by ~gen 100 as the control's own clonal fixation catches up, hence the 28-generation scope. The co-evolved code remains a **private kin code** rather than a public convention: the survival arm fixates to a single founder lineage (dominant-lineage share 99.1%, N_eff 1.02), within-founder mutual intelligibility reaches ≈0.16 (about 100× chance) while cross-founder intelligibility sits at chance. When diversity is actively sustained (niching at pop=96, soft96, N_eff 4.10 over 20+ generations), cross-founder intelligibility still does not leave the floor (directional at n=8; CF−FLOOR CI straddles zero). Maintaining diversity is therefore necessary but not sufficient. Across **three heterogeneous interventions** (C2X, C2X2, C2X3 — different designs and scaffolds, 40 vs 48 generations) that triangulate cross-lineage pressure, cross-founder intelligibility *never rose above chance*; where the pressure bit, it only crashed survival harder, because non-kin codes are mutually undecodable and forcing non-kin listening starves foraging — a **pressure-vs-survival tension**. We scope every part of this to *this world and this architecture*, and to each stage's timescale: the headline co-evolution, kin-only diagnostic, and C1 arms run 28 generations, while the cross-lineage probes run 40–48 generations. Under these regimes a public code *did not* bootstrap; we explicitly *decline to bank this as a formal negative* (n=8 positive-or-inconclusive tier; a formal negative would want n=16).

**Methodologically**, this arc is more believable because of the bidirectional self-audit that produced it. Its single claim is **bidirectionality** — catching deflation as well as inflation — anchored on two in-scope, reproducible catches: a pilot/formal false *negative* (§5.1) and an RNG-offset bug in the headline random-fitness control surfaced by a multi-model adversarial review (§4.2), whose fix left the conclusion intact and slightly *strengthened* the margin (+0.045 → +0.0548). Routine in-scope episodes (a frozen gate held to an honest inconclusive, contradictory reviewers adjudicated, an over-coarse auto-label corrected via a pre-logged continuous metric) show the discipline in normal operation. We present it as a practice we recommend, not as a co-equal scientific result.

**Future work.** The arc is self-contained; it *motivates rather than requires* its sequel. The central next question is sharp: **is the public-code bottleneck the world/ecology, or the optimizer?** This paper's cross-lineage null (CF at floor across three interventions) plus the transient finding (selection only *accelerates* a clonal-descent channel) motivate that program but do not decide it. We name two candidate interventions, and require both to be **oracle-clean and not gradient-on-survival** so the result cannot be trivialized:

(a) **A richer world.** Spatial demes plus a resource ecology that *naturally* sustains multiple coexisting lineages (so diversity need not be propped up by a quota that, as in C2X3, fails to hold N_eff anyway), complementary/legal local perception, and a **real survival payoff for decoding non-kin** — so that cross-lineage understanding actually pays its metabolic rent rather than starving foraging.

(b) **Cultural transmission / lifetime learning that is consequence-driven.** Agents learn within their lifetime from their *own* survival outcome, or imitate conspecific utterances — but are **never supervised on the true referent** (which would be an oracle and would trivially manufacture a shared code). Neural iterated learning is an established from-scratch route to structured languages under an imposed transmission bottleneck (Ren et al., 2020); the open problem here is doing that under survival selection in a grounded world, without borrowing a pre-trained language or leaking the true referent. The project's existing oracle-clean cultural-imitation (Kirby) mechanism already shows a within-population null, so crossing the **kin wall** with a consequence-driven learner is a genuinely open problem, not a foregone conclusion.

We frame this paper's cross-lineage null and the transient acceleration as *motivating* that next program: whether a decodable code stays kin-scoped once the world sustains diversity and rewards cross-lineage decoding, or once a consequence-driven learner can re-couple drifting lineages, is the open question this work was built to pose, not to answer.

### Seeds, configs, and determinism

Every run is seeded; the same command at the same commit yields a byte-identical verdict. Seed counts are recorded in the verdict JSONs and ledger: the cross-lineage tier (C2X / C2X2 / C2X3) is n=8, and the frozen tier rule permits only POSITIVE or INCONCLUSIVE at n≤8 — a scoped negative requires n=16. C2X3 supports seed-parallelism via `C2X3_WORKERS=N` (default 1; capped ~6 on a 16-core box, verified byte-identical to serial). Re-running overwrites the working file; the banked version stays in git at the ledger-cited commit. Newer runners embed a `provenance` block (`make_provenance` → `command_hint`, `env`, `effective_config`, `git_commit`, `git_dirty`); pre-feature g1f-arc verdicts are reproducible via the runbook and ledger citations (by verdict filename, not commit hash).

### CI invariants

Heavy runs cannot be CI-automated (GPU/CPU minutes-to-hours); they are version-controlled *recipes*, not pipeline steps. What *is* automated are the scale-independent harness invariants under `tests/`, enforced on every push and pull request by `.github/workflows/ci.yml` via `python -m pytest tests/ -q`. These guard against silent behavior drift: vectorized MII is byte-identical to the reference loop (including a deliberate argmax tie); `_select_next_quota(K=0)` equals `_select_next_soft`; adding `speaker_rule=` leaves the default path byte-stable; forced cross-lineage routing never returns the listener's own lineage; the shared helpers behave as the runners expect; `make_provenance` produces a well-formed record; all RTC modules import cleanly; and the no-oracle redline scan (`_redline_scan` over `rtc_g1f_coevolve.py`) returns zero hits for out-of-`_run_episode` `world.patch` access and for the three banned training calls (`cotrain_factored`, `train_relay_channel`, `find_nearest`). This automated scan covers the core co-evolution runner only; the cross-lineage runners' redline compliance (and the edibility/target-code/founder-id-as-input prohibitions) is enforced by construction and checked by code-grounded review (§4.6), not by this scan.
